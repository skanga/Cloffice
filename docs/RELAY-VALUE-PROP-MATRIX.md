# Relay Value Proposition Matrix

Date: 2026-03-23
Status: Draft v1

## 1. Founder or CEO
Primary pains:

- Constant context switching.
- Slow decision loops.
- Inconsistent execution quality across team members.

What Relay delivers:

- Daily and on-demand operational synthesis.
- Faster decision-ready recommendations.
- Controlled delegation with approval gates.

Proof metrics:

- Decision latency reduction.
- Fewer founder interrupts per week.
- Higher throughput per operator.

## 2. Operations Lead
Primary pains:

- Work spread across too many tools.
- Manual coordination and follow-ups.
- Weak visibility into run state and ownership.

What Relay delivers:

- One cowork thread that spans systems.
- Structured plan-approve-execute flow.
- Shared run timeline, ownership, and outcomes.

Proof metrics:

- Cycle time per workflow.
- Completion rate of end-to-end runs.
- Reduced manual handoff time.

## 3. Finance Lead
Primary pains:

- Approval bottlenecks.
- Policy inconsistency.
- Audit pressure and evidence collection burden.

What Relay delivers:

- Policy-aware recommendation states.
- Approval and escalation routing.
- Receipts and exportable audit history.

Proof metrics:

- Approval SLA improvement.
- Exception handling time.
- Audit preparation time reduction.

## 4. Product or Customer Insights Lead
Primary pains:

- Feedback spread across channels.
- Hard to prioritize signal vs noise.
- Slow translation from insight to roadmap action.

What Relay delivers:

- Cross-source synthesis with rationale.
- Theme clustering with priority logic.
- Actionable output for planning cycles.

Proof metrics:

- Time-to-insight.
- Time-to-prioritized recommendation.
- Stakeholder confidence in findings.

## 5. IT or Security Stakeholder
Primary pains:

- Shadow AI usage.
- Unclear data flows and weak controls.
- Governance debt from disconnected pilots.

What Relay delivers:

- Centralized orchestration on OpenClaw.
- Explicit controls for approval and execution.
- Traceable event history by run.

Proof metrics:

- Reduction in unmanaged AI workflows.
- Percentage of runs with full audit trace.
- Incident and exception trend quality.

## 6. Team-Level Summary
Common value across roles:

1. Less switching, more execution.
2. Human control over consequential decisions.
3. Shared context and standardized operating patterns.
4. Measurable outcomes tied to business workflows.

## 7. Packaging Narrative by Stage
Pilot stage:

- One high-value recipe, clear baseline metrics, tight support loop.

Expansion stage:

- Add adjacent recipes in same governance model.

Enterprise stage:

- Multi-team rollout with standardized controls, audit, and policy operations.
